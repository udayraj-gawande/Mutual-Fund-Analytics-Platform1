# 📁 data/

This folder contains all raw and processed datasets used by the Mutual Fund Analytics Platform.

---

## Folder Structure

```
data/
├── raw/          ← Original unmodified source CSV files
├── processed/    ← Cleaned & enriched CSVs produced by the ETL pipeline
└── db/           ← (Optional) SQLite exports or query results
```

---

## Raw Data (`data/raw/`)

> Files populated by downloading from AMFI / MF API or provided as seed data.

| File | Description |
|------|-------------|
| `01_fund_master.csv` | Master list of mutual fund schemes with AMFI codes and categories |
| `02_nav_history.csv` | Daily NAV (Net Asset Value) history across all schemes |
| `03_aum_by_fund_house.csv` | AUM (Assets Under Management) aggregated by fund house |
| `04_monthly_sip_inflows.csv` | Month-wise SIP inflow data across the industry |
| `05_category_inflows.csv` | Net inflows broken down by fund category |
| `06_industry_folio_count.csv` | Total investor folios by fund house |
| `07_scheme_performance.csv` | Performance metrics (return, Sharpe ratio, expense ratio, etc.) |
| `08_investor_transactions.csv` | Simulated investor transaction-level dataset |
| `09_portfolio_holdings.csv` | Portfolio-level stock holdings per fund |
| `10_benchmark_indices.csv` | Benchmark index data (Nifty 50, Sensex, etc.) |

Additionally, `live_nav_fetch.py` saves live-fetched NAV files here:
- `sbi_bluechip_nav.csv`
- `icici_bluechip_nav.csv`
- `nippon_large_cap_nav.csv`
- `axis_bluechip_nav.csv`
- `kotak_bluechip_nav.csv`

| File | Source |

| `clean_nav_history.csv` | Cleaned `02_nav_history.csv` with parsed dates |
| `clean_aum_by_fund_house.csv` | Cleaned `03_aum_by_fund_house.csv` |
| `clean_monthly_sip_inflows.csv` | Cleaned `04_monthly_sip_inflows.csv` |
| `clean_category_inflows.csv` | Cleaned `05_category_inflows.csv` |
| `clean_industry_folio_count.csv` | Cleaned `06_industry_folio_count.csv` |
| `clean_scheme_performance.csv` | Cleaned `07_scheme_performance.csv` |
| `clean_investor_transactions.csv` | Cleaned `08_investor_transactions.csv` |

---

## Notes

- **Do not manually edit** files in `processed/` — they are regenerated on every pipeline run.
- Raw files are considered immutable source data. Update them only when sourcing new data.
- Run `python run_pipeline.py` from the project root to refresh all processed files.
